# Private Markets Marketplace
## Design System (v1.0 – Institutional)

## 1. Design Philosophy

### Primary Goal:
Communicate trust, control, permanence, and regulatory seriousness.

### Secondary Goals:
- High information density
- Zero visual noise
- Clear statefulness (approvals, statuses, locks)
- UI that does not emotionally influence trading behavior

**If a compliance officer is comfortable using this daily, the design is correct.**

**Rule:** No pure red or green. Everything is toned down.

## 3. Typography System

### 3.1 Font Stack (Recommended)

#### Primary Font (UI & Tables)
**Inter**

```css
font-family: 'Inter', system-ui, -apple-system, BlinkMacSystemFont;
```

**Why:**
- Neutral
- Excellent legibility at small sizes
- Widely trusted in financial systems

#### Optional Heading Accent (Later Phase)
**Source Serif 4** – optional
- Only for large section headings
- Not required for v1

### 3.2 Type Scale

| Token | Size | Weight | Usage |
|-------|------|--------|-------|
| text.xl | 20px | 600 | Page titles |
| text.lg | 16px | 600 | Section headers |
| text.md | 14px | 500 | Body text |
| text.sm | 12px | 500 | Metadata |
| text.xs | 11px | 500 | Table footnotes |

**Line height:** 1.45–1.6

**No decorative text. Ever.**

### 4.2 Layout Rules
- Left navigation, fixed
- Content max width: 1280px
- Tables over cards where possible
- Forms grouped in bordered sections

## 5. Component System

### 5.1 Buttons

| Type | Style |
|------|-------|
| Primary | Solid accent |
| Secondary | Outline |
| Tertiary | Text only |

**Rules:**
- Border radius: 4px
- Height: 36px
- No pill buttons
- No icon-only primary buttons

### 5.2 Tables (Critical Component)
Tables are first-class citizens.

**Design Rules:**
- Zebra striping (very subtle)
- Sticky headers
- Right-aligned numbers
- Explicit column headers

**Example:**
- Marketplace listing
- Tranches list
- Approval queues

### 5.3 Forms
- Labels above fields
- Required fields explicitly marked
- Inline validation (no toasts)
- Grouped in panels

### 5.4 Status Indicators
- Text + muted chip
- Never color-only
- **Example:** Approved, Pending Review

## 6. Navigation

### 6.1 Primary Navigation (Left)
- Marketplace
- Private Securities
- Orders
- RFQs
- Clients
- Compliance (role-based)
- Admin (role-based)

Icons are optional; text first.

## 7. Document & Data Room Design
Documents must feel controlled.

- **Table view**
- **Columns:** Name | Type | Uploaded By | Date
- Download is explicit (icon + text)
- No previews in v1

## 8. Motion & Effects
- Strictly minimal
- 150–200ms transitions
- No easing theatrics
- No loading spinners → use skeletons

## 9. Accessibility & Readability
- WCAG AA contrast minimum
- Keyboard navigable tables
- Clear focus states
- No reliance on color alone
